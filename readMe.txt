本次作业的任务是优化这个在汇编语言下的申请和释放内存，我主要在两个方面进行了优化：
1，减少了系统调用brk的次数。
2，在释放内存的时候，合并临近的available的内存块。

我主要通过一下方法进行的优化：
1，在每次应用brk调用的时候，向系统申请512的整数倍的内存，申请更多的空闲内存来减少brk的调用次数。
2，在每个数据块的结尾加上跟前面的header一样的标签，另其方便向前合并。
3，在创建新的数据块的时候，切割原大数据块，使其变成两个小数据块，并将其中一个给程序用。
4，在删除的时候，分别向前和向后搜索是否有available的数据块，有的话，更改两端的标签，使其合并为一个数据块。

我用c语言进行调用测试了数据：
首先分别申请6,513,40的内存，然后一次删掉第2,1,3个指针，然后在申请800和2000的内存。

程序的编译方法为：
我的程序为：
gcc alloc.c alloc1.s -o alloc1 -m32
用 ./alloc1 运行
ppt为：
gcc alloc.c alloc.s -o alloc -m32
用 ./alloc 运行

ppt程序的结果为：
p1: 0x9b21008
p2: 0x9b21016
p3: 0x9b2121f
p4: 0x9b2124f
p5: 0x9b21577
The time is 71.000000

我的程序结果为：
p1: 0x8aec008
p2: 0x8aec01e
p3: 0x8aec22f
p4: 0x8aec008
p5: 0x8aec418
The time is 70.000000

可以看出通过更多的调用空间和数据块合并，可以减少brk次数，并且有效的利用已经释放的空间。在速度方面，由于搜索复杂度仍然是O(n)，所以速度没有明显提升。
